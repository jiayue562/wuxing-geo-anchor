### 1.3 AI加工详细流程

```markdown
【第一步：剪藏】

Web Clipper → 选择目标仓库 → 点击保存
→ 自动转换为Markdown格式

【第二步：AI深度加工】

调用DeepSeek API → 输入剪藏内容
→ 系统自动生成：

1. 一句话总结
   格式：基于[主题]，提出[核心观点]

2. 关键点列表
   格式：3-5个Bullet Points
   内容：每个点不超过20字

3. 价值评价
   格式：[rating:: 1-5]
   说明：适用场景、局限性

4. 标签建议
   格式：#标签1 #标签2
   规则：最多5个标签

【第三步：初始标记】

为所有新笔记添加：#未归档
→ 便于后续集中整理
```

### 3.2 对比与交叉阅读

```markdown
【什么是交叉阅读】

同时打开2-4篇相关笔记进行对照阅读
- 观点互补的笔记
- 观点相反的笔记
- 看似无关但可能关联的笔记

【为什么有效】

并置阅读 ──→ 强制大脑寻找 ──→ 触发
                  ↓
            模式 + 差异 + 联系
                  ↓
            新想法 + 新问题
                  ↓
            写作和创造的直接源泉

【操作步骤】

1. 找到相关笔记（本地图谱/反向链接）
2. 同时打开2-4篇
3. 并排显示
4. 记录差异和联系
5. 生成新想法
```

### 3.3 图谱查漏补缺

```markdown
【每周图谱复盘】

1. 打开全局图谱
2. 寻找孤立节点
3. 分析原因：
   - 为什么孤立？
   - 应该与哪个概念连接？
   - 需要创建什么主题关联？

4. 执行行动：
   - [[孤立笔记]] ──→ [[某个概念]]
   - 或创建新的综述笔记
   - 或合并到已有主题

【目标】
修复和强化知识网络的结构完整性
```
