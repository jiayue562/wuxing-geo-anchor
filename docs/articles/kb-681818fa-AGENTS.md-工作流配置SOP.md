## 二、Ingest SOP：入库操作

### 触发词
- 「处理一下Notes/」
- 「把这个整理进知识库」
- 「Ingest」

### 操作流程

#### Step 1: 列方案
扫描Notes/下所有未处理的文件（Clippings/Inbox/Conversation），列出一个表：

```markdown
| 文件 | 建议放哪 | 怎么处理 | 为什么 |
|------|---------|---------|--------|
| xxx.md | Knowledge/ | 新建 | 全新主题 |
```

**分类规则**：
| 内容性质 | 目标目录 |
|---------|---------|
| 软件工具、代码技巧 | Software/ |
| 投资、保险、健康 | LifeOS/ |
| 方法论、读书笔记、概念 | Knowledge/ |
| 内容创作方法论 | Writing/Knowledge/ |
| 选题素材 | Writing/ |
| 判断不了 | 留在Notes/，标注待确认 |

#### Step 2: 等待确认
列出方案后等待用户确认。确认前不执行任何写操作。

#### Step 3: 加工
- 读原文提取关键信息
- 写摘要（摘要格式见附录）
- 在目标目录创建或更新页面
- 给相关页面补交叉引用

#### Step 4: 清理
- 原始文件从Notes/删除
- 确认目标页面内容正确


## 四、Lint SOP：健康检查

### 触发词
- 「跑一下健康检查」
- 「Lint」
- 定期提醒（建议每月一次）

### 检查范围

按目录分别跑，不一次全跑：

| 目录 | 重点 | 适合Lint? |
|------|------|-----------|
| Knowledge/ | 链接完整性、内容过时 | ✅ 重点 |
| LifeOS/ | 投资笔记时效性 | ✅ 选择性 |
| Software/ | 工具版本是否过时 | ⚠️ 效果有限 |
| Writing/ | 创作素材完整性 | ✅ 有价值 |

### 四项检查

1. **孤立页面检测**：没有任何入链的页面
2. **过时内容检测**：超过90天未更新的页面
3. **缺失引用检测**：讲同一概念但未链接的页面
4. **Notes/积压检测**：超过7天未处理的内容

### 输出格式

```markdown
## 健康检查报告 [目录名]

### 孤立页面: N 个
- [页面A] — 建议: 链接到相关页面

### 过时内容: N 个
- [页面B] — 最后更新: YYYY-MM-DD — 建议: 检查是否需要更新

### 缺失引用: N 对
- [页面C] ↔ [页面D] — 内容互补，建议建链接

### Notes/积压: N 个
- [文件E] — 等待处理超过7天
```

跑完出报告，不自动改。
