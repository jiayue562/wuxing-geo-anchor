# 🤖 岗位智能体 · SKILL.md

> **核心定位**：基于五行人格心理学OS的岗位AI工作流设计系统  
> **一句话功能**：通过"行业×企业×岗位×性格×任务×模型"六维框架，生成精准适配的AI提示词与智能体使用方案  
> **版本**：v3.0 | **创建日期**：2026-04-20 | **更新日期**：2026-05-18 | **维护者**：龙龟神将


## 🎮 双模式 + 三种输出类型

> **核心升级**：岗位智能体v3.0支持两种模式、三种输出类型。

### 模式 A：👤 提示词模式（默认）
- **输出**：人用 AI 提示词
- **适用**：给企业员工设计 AI 协作助手
- **示例**：味藏总经理的 AI 助手提示词

### 模式 B：🐉 龙爪模式
- **输出类型 B1**：个人复盘画布HTML（聚焦个人一周工作的6步深度分析）
- **输出类型 B2**：组织诊断报告HTML（聚焦组织7S+经营+文化的多维度诊断）
- **适用**：给 WorkBuddy 造龙爪（AI 智能体）
- **示例**：味藏店长周复盘画布（B1）/ 堂食创业单元7S诊断报告（B2）

### 模式切换指令

| 你想做什么 | 输入 |
|-----------|------|
| 给人设计提示词 | 默认模式，直接按六维框架输入 |
| 造 WorkBuddy 龙爪 | 先说"切换到龙爪模式"，再输入设计 |

### 报告类型判断逻辑（v3.0新增）

```yaml
报告类型自动判断:
  输入包含个人名 + 周数据 → B1个人复盘画布
  输入包含多门店/多单元数据 → B2组织诊断报告
  输入仅有岗位描述 + 任务 → 提示词模式（默认）
```


## 🎯 使用流程

### Step 1: 六维信息采集

**输入模板**：

```yaml

  行业: [餐饮/科技/教育/金融等]
  企业简介: [规模/阶段/文化特色]
  岗位: [市场总监/总经理/运营经理等]
  性格: [水行人-灵活/火行人-热情/木行人-创新/金行人-目标/土行人-稳定]
  核心任务: [渠道投放/用户增长/品牌策划等]
  适用模型: [GROW/OKR/第一性原理/五色光方法论等]
```

**示例**：
```yaml
六维框架示例:
  行业: 餐饮行业，日料，金枪鱼专门店
```

#### 数据驱动五行识别机制（v3.0新增）

> **触发条件**：用户未提供五行类型，或用户提供的数据与自报类型矛盾时自动启动。

```yaml
数据驱动五行识别机制:
  触发条件: 用户未提供五行类型 或 用户提供的数据与自报类型矛盾
  识别逻辑:
    - 目标导向+执行快+数据驱动 → 金行信号
    - 热情高+感染力强+跟进弱 → 火行信号
    - 创新力+前瞻性+落地弱 → 木行信号
    - 稳健+流程强+变革慢 → 土行信号
    - 灵活+策略强+定力弱 → 水行信号
  输出: 推断的五行类型 + 识别依据 + 阴阳面标签
  
  示例:
    数据: 金月湾完成率148.8%，超额目标，执行极快
    推断: 金行人（目标强执行快）
    阳面标签: ✅目标感强 ✅执行力强 ✅数据意识强
    阴面标签: ❌关系维护弱 ❌文化把关弱 ❌过程指标少
    
    数据: 松山湖渗透率50%，销售热情高
    推断: 火行人（热情高感染力强）
    阳面标签: ✅客户链接力强 ✅行动力快
    阴面标签: ❌跟进弱 ❌系统性不足
```

### Step 3: 7S组织诊断评分（v3.0新增）

> **定位**：在五行识别之后、关键动作设计之前，用7S框架对组织进行定量诊断。

```yaml
7S评分标准:
  S1战略 (Strategy):
    8-10: 战略清晰且可执行，团队认知统一
    5-7: 有战略但执行走样或认知不统一
    1-4: 战略模糊或与实际脱节
  S2结构 (Structure):
    8-10: 组织架构清晰，权责明确，协作流畅
    5-7: 基本架构在但存在模糊地带
    1-4: 架构缺位或权责混乱
  S3制度 (Systems):
    8-10: 制度完善，流程标准化，执行一致性高
    5-7: 有制度但执行不统一或覆盖不全
    1-4: 制度缺位或形同虚设
  S4风格 (Style):
    8-10: 管理风格鲜明且适配组织，团队认同度高
    5-7: 有管理风格但不一致或不适配
    1-4: 管理风格混乱或专制
  S5人员 (Staff):
    8-10: 人才梯队完整，关键岗位无空缺
    5-7: 核心岗位有人但梯队不完整
    1-4: 关键岗位空缺或人岗不匹配
  S6技能 (Skills):
    8-10: 核心技能完备，持续培训体系运转
    5-7: 有基本技能但缺乏深度培训
    1-4: 关键技能缺失，无培训体系
  S7共同价值观 (Shared Values):
    8-10: 价值观清晰，全员认同，日常行为一致
    5-7: 有价值观但认知不统一或行为偏离
    1-4: 价值观模糊或仅停留在口号

7S诊断依据模板:
  维度: [S1-S7]
  评分: [1-10]
  诊断依据: [基于数据/行为的具体证据]
  核心问题: [一句话概括瓶颈]
  改善方向: [具体可执行的改善建议]

7S可视化规范:
  评分8-10: 绿色进度条 + "强/良"标签
  评分5-7: 橙色进度条 + "中/偏弱"标签
  评分1-4: 红色进度条 + "弱"标签
  综合评分: 7维加权平均（默认等权，可自定义权重）

7S诊断输出示例:
  S2结构:
    评分: 4
    诊断依据: "组织架构调整连续两周未完成，权责不清；三店执行差异巨大（完成率93.9%~148.8%）"
    核心问题: "权责不清导致执行割裂"
    改善方向: "明确三店店长权责边界，建立双周组织架构检视机制"
```

### Step 4: 三层指标体系 + 关键动作设计（v3.0升级）

> **v2.0→v3.0变化**：将"目标量化/数据验证/风险控制"重组为三层指标金字塔，新增P0/P1优先级、4列风险预警表、分层关联规则。

```yaml
三层指标体系:
  Layer 1 - 结果指标（考核用·绿色）:
    定义: 衡量最终经营成果的指标
    模板: 指标名 | 当前值 | 目标值 | 达成状态
    示例: 综合营业额完成率 | 132.9% | 100% | 达标
    频率: 月度/季度
    颜色编码: 绿色(tag-g) = 达标/完成/优势

  Layer 2 - 过程指标（监控用·蓝色）:
    定义: 衡量执行过程质量的指标
    模板: 指标名 | 当前值 | 预警阈值 | 监控频率
    示例: 1对1邀约执行率 | 极低 | 每店每日≥3人 | 日度
    频率: 日度/周度
    颜色编码: 蓝色(tag-b) = 过程/监控/信息

  Layer 3 - 风险指标（预警用·红色）:
    定义: 需要立即关注的系统性风险
    模板: 指标名 | 描述 | 严重度 | 优先级
    示例: 客户链接机制断裂 | 三店系统性失效 | 系统级 | P0
    频率: 实时
    颜色编码: 红色(tag-r) = 预警/未达标/薄弱

  分层关联规则:
    - 过程指标连续3期低于阈值 → 升级为风险指标
    - 风险指标P0 → 当周必须解决，纳入每日行动日历
    - 风险指标P1 → 当月必须解决，纳入每周复盘

关键动作设计:
  资源分配:
    预算分配:
      - 类别1: [占比] - [用途]
      - 类别2: [占比] - [用途]
    人力配置:
      - 岗位1: [人数] - [职责]
    时间规划:
      - 阶段1: [时间] - [里程碑]

  风险控制（v3.0升级为4列）:
    模板: 下周任务 | 潜在风险 | 预警信号 | 应对预案
    预警信号定义: 必须是可观测、可量化、有时效的现象
    任务锚定: 每个风险必须关联到一个具体任务
    示例:
      - 520营销活动 | 客流超预期 | 11:30前等位超5桌 | 启动弹性补位+简化流程
      - 新品上市 | 供应链断裂 | 核心食材库存<3天 | 启动备选供应商
```

#### 选项 B1：🐉 个人复盘画布模板（v3.0新增）

> **聚焦**：一个店长/管理者的一周工作深度复盘

```yaml
B1模板结构:
  Step A: 工作全量盘点
    输入: 本周完成事项清单
    处理: 每项标注7S维度标签
    输出: 带7S标签的工作盘点表 + 完成率进度条

  Step B: 五行优劣势剖析
    输入: 五行类型 + 工作盘点数据
    处理: 阳面=五行显发优势（附行为证据），阴面=过刚/过柔劣势（附行为证据）
    输出: 双栏对比（优势5项 vs 劣势5项）

  Step C: 原因剖析（根因链条）
    输入: 阴面劣势项
    处理: 每个劣势追溯因果链（至少3层），直到可执行的根因
    输出: 根因链条卡片（起因→过程→结果）

  Step D: 拔阴取阳行动方案
    输入: 根因链条 + 五行类型
    处理: 阴面→取阳方向→具体行动→借力五行
    输出: 4列表格（阴面/取阳方向/具体行动/借力五行）

  Step E: 下周风险预警
    输入: 下周任务 + 阴面劣势
    处理: 每个任务识别风险→预警信号→应对预案
    输出: 4列风险预警表（下周任务/潜在风险/预警信号/应对预案）

  Step F: 进化路径
    输入: 当前优势 + 阴面改善方向
    处理: 设计3-4级进化阶梯（当前→目标），每级有核心关键词
    输出: 进化路径可视化 + 当前定位 + 目标定位

  一周一句话:
    模板: "[人物]是[组织]最[五行意象]的[隐喻]——[阳面成就]。但[五行意象]需要[借力五行]来[转化方向]。[核心行动方向]，不是做更多，而是做更深。"
```
