# 龙爪模式

> **核心定义**：岗位智能体v2.0新增的第二种输出模式，用于生成WorkBuddy SKILL.md + 可选Rule .mdc

## 与提示词模式对比

| 维度 | 提示词模式 | 龙爪模式 |
|------|-----------|---------|
| 输出 | 人用AI提示词 | WorkBuddy SKILL.md + Rule |
| 适用 | 给企业员工设计AI协作助手 | 给WorkBuddy造龙爪（AI智能体） |
| 触发 | 默认模式，直接输入 | 先说"切换到龙爪模式" |
| 流程 | 6步 | 5步 + 选项B分支 |

## 龙爪模式工作流

```
六维输入 → 五行识别 → 关键动作 → AI关键词 → 选择输出类型
  └→ 选项B：输出 SKILL.md + 资产引用 + 可选 Rule
```

## SKILL.md模板结构

1. YAML frontmatter（title、description、version、created、tags、alwaysApply）
2. 概念澄清：[[智能体vsSkill]]
3. What（是什么）
4. Why（为什么）
5. How（怎么做）
6. 核心理论体系
7. 核心金句精选
8. 标签系统
9. 关联文档与扩展阅读
10. 版本历史

## Rule文件（.mdc）结构

1. YAML frontmatter（name、description、version、created、tags、alwaysApply）
2. 核心定位
3. 触发条件（两层）
   - 第一层：上下文感知触发（无需关键词·最高优先级）
   - 第二层：直接触发词（P0系统词权重5 + P1场景词权重4）
4. 激活后的行为（MUST FOLLOW）
5. 关联Skills路径

## 资产引用规则

龙爪模式下，根据用户场景自动填充资产引用路径：
- 五行人格分析类 → [[五行总智能体]]、[[凤心OS]]、[[凤脑OS]]
- 味藏/餐饮类 → [[味藏顶层设计]]、[[岗位智能体]]
- 企业类 → [[企业文化OS]]、[[岗位智能体]]
- 沟通/关系类 → [[凤爪OS]]、[[心文化]]
- 通用类 → [[龙心OS]]、[[龙脑OS]]、[[龙爪OS]]
