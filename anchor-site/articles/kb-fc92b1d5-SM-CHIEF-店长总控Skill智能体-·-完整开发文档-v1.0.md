## 核心定位与设计哲学

### 1.1 为什么需要店长总控Skill智能体？

```
┌─────────────────────────────────────────────────────────────┐
│                    三重痛点诊断                              │
├─────────────────────────────────────────────────────────────┤
│ ① 信息过载：店长每天被数据淹没，1.5h日反馈耗时严重         │
│ ② 判断分散：7个信息交会点缺乏系统化支撑，靠经验拍板       │
│ ③ 成长瓶颈：从"事务管理者"升维到"战略经营者"缺少数字引擎   │
│                                                             │
│ ④ 文化断层：董事长文化意图→总经理战略→店长执行存在衰减     │
│ ⑤ 数据断裂：日反馈→周复盘→月度会议→战略校准缺乏连续性     │
│ ⑥ 人才断层：店长能力无法复制，依赖个人英雄主义           │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心设计哲学（三源合一）

| 维度 | 董事长视角（木行人） | GM视角（金行人） | 店长视角（实战） |
|------|-------------------|---------------|---------------|
| **本质** | 愿景牵引·文化布道 | 执行穿透·结果交付 | 问题解决·效率提升 |
| **核心问题** | "这件事是否符合'成就有才（财）之人'？" | "这件事能否达成36,049,064.23元？" | "今天怎么做最有效？" |
| **价值主张** | 店长是文化的"最后一公里"传播者 | 店长是战略的"第一公里"执行者 | 店长是团队的"每一天"经营者 |
| **AI侧重** | 文化渗透率检测+价值观行为化追踪 | 战略解码准确性+KPI达成率预警 | 日效率释放+判断辅助+流程自动化 |
| **输出物** | 文化健康度评分+仪式践行报告 | 战略看板+经营仪表盘+异常预警 | 结构化日卡片+行动建议P0/P1/P2 |

### 1.3 三角协同架构

```
                    ┌─────────────────┐
                    │   CHMN-BRAIN    │  ← 董事长层（愿景·文化·价值观）
                    │      ↓ 调度       │
                    │   GM-CHIEF      │  ← 总经理层（战略·解码·闭环）
                    │      ↓ 赋能       │
                    │  SM-CHIEF ★     │  ← ★ 店长总控层（本文档主题）
                    │      ↓ 调度       │
          ┌───────────┼───────────┬───────────┐
          ↓           ↓           ↓           ↓
      [N-D01]    [N-D02]    [N-W01]    [M01]
       晨间启动    营运管理    周复盘     月度会

      [日反馈★]   [P-01]     [五行适配]  [7S映射]
       智能体     路由引擎    差异化配置   全维穿透
```


### Step 2：信息交会点7个 → AI边界铁律

#### 2.1 七大交会点时序图谱

| # | 时间 | 交会点名称 | 你的判断 | AI做的事 | 输出物 |
|---|------|-----------|---------|----------|--------|
| **J01** | 10:00 | **食材验收判断** | 最终验收签字 | 推送收货清单+异常标记(颜色/份量/新鲜度) | 验收确认卡 |
| **J02** | 10:00 | **今日重点判断** | 确认优先级 | 汇总昨日数据+异常标记+遗留问题 | 今日重点清单 |
| **J03** | 11:30 | **巡检判断** | 逐一确认打钩 | 巡检模板+上次问题提醒+照片对比 | 巡检完成卡 |
| **J04** | 12:00-13:30 | **观察判断** | 感知+判断 | 实时监控+记录入口(员工状态/话术/客人表情) | 员工观察日志 |
| **J05** | 14:00-17:00 | **辅导判断** | 决定是否谈/怎么谈 | 待办+面谈建议+历史记录+超期提醒 | 面谈建议卡 |
| **J06** | 21:00 | **问题分级判断** | 确认分级 | 自动填充+分类建议(1级组长/2级店长/3级公司) | 问题分级报告 |
| **J07** | 21:30 | **夕会校验判断** | 校验确认 | 问题清单+明日初稿+预审+遗漏扫描 | 夕会校验卡 |

#### 2.2 AI边界铁律（六大原则）

| AI做（预处理层） | 人做（决策层） | 设计原理 |
|------------------|--------------|---------|
| 信息预处理（数据→选择题） | 现场判断（温度/情绪/细节） | AI降低认知负荷，人聚焦高价值判断 |
| 模板提供（巡查/面谈/反馈标准版） | 拍板决策（辞退/重大调整/例外处理） | AI提供结构，人保留自由裁量权 |
| 自动流转（问题→责任人+时限） | 人际沟通（重要客户/冲突协调/情感支持） | AI管事不管人 |
| 进度追踪（谁还没闭环/超时预警） | 亲身辅导（组长能力培养/文化传递） | AI盯进度，人投成长 |
| 差异对比（今天vs昨天/环比/同比） | 战略直觉（市场变化/机会捕捉/风险预感） | AI给数据，人给方向 |
| 异常预警（红线触碰/KPI偏离/趋势恶化） | 最终裁决（是否升级/是否启动PIP） | AI报警，人拍板 |

**G-SMC-J01**："AI不做判断，AI只让判断更快更准——这就是SM-CHIEF的存在意义"
