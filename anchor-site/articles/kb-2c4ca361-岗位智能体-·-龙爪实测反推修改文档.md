## 🔬 反推方法

```
输入：两份龙爪实测HTML（狮山店周复盘画布 + 堂食创业单元7S诊断报告）
  → 对照：岗位智能体v2.0的8步SOP + 两种输出模板
  → 识别：龙爪实际输出中哪些内容是岗位智能体无法生成的
  → 归因：定位Skill层面的缺失（流程缺步/模板缺项/框架缺维）
  → 输出：13项修改清单（含优先级/修改原因/具体内容/预期效果）
```

**核心发现**：岗位智能体v2.0的8步SOP覆盖了"从输入到输出骨架"的流程，但在以下三个层面存在系统性缺口：

1. **深度分析框架**（7S评分、指标分层、五行阴阳面）
2. **多维度诊断输出**（复盘画布、龙爪断语）
3. **视觉化呈现规范**（HTML仪表盘设计）

本次升级从v2.0→v3.0，核心目标是**让岗位智能体能够直接生成龙爪级别的深度诊断报告**。


### P0-1：新增7S评分体系与诊断依据模板

**⚡ 龙爪实测证据**

堂食创业单元7S诊断报告中，7S维度各有**1-10分评分 + 进度条 + 评级标签（强/良/中/弱）+ 诊断依据文字 + 核心问题 + 改善方向**。例如S2结构评4分，诊断依据为"组织架构调整连续两周未完成，权责不清；三店执行差异巨大（完成率93.9%~148.8%）"。

当前岗位智能体Step 2仅有"五行-岗位适配表"，**完全没有7S评分方法论**。SKILL.md中7S仅在龙爪模式输出模板中被提及为"核心理论体系"，但无评分标准、无诊断依据模板、无评级逻辑。

**🔧 具体修改**

**新增 Step 2.5：7S组织诊断评分**（插入在五行识别之后、关键动作设计之前）

1. 新增7S评分标准表（1-10分制，每分对应的行为描述）
2. 新增7S诊断依据模板：评分 + 诊断依据 + 核心问题 + 改善方向
3. 新增7S可视化规范：进度条颜色（8-10绿/5-7橙/1-4红）+ 评级标签逻辑
4. 新增7S综合评分计算方法：7维加权平均（默认等权，可自定义权重）

```yaml
# 新增 Step 2.5：7S组织诊断评分
7S评分标准:
  S1战略:
    8-10: 战略清晰且可执行，团队认知统一
    5-7: 有战略但执行走样或认知不统一
    1-4: 战略模糊或与实际脱节
  S2结构:
    8-10: 组织架构清晰，权责明确，协作流畅
    5-7: 基本架构在但存在模糊地带
    1-4: 架构缺位或权责混乱
  # ... S3-S7 类似标准

7S诊断依据模板:
  维度: [S1-S7]
  评分: [1-10]
  诊断依据: [基于数据/行为的具体证据]
  核心问题: [一句话概括瓶颈]
  改善方向: [具体可执行的改善建议]

7S可视化规范:
  评分8-10: 绿色进度条 + "强/良"标签
  评分5-7: 橙色进度条 + "中/偏弱"标签
  评分1-4: 红色进度条 + "弱"标签
  综合评分: 7维加权平均（默认等权）
```

**📈 预期效果**

岗位智能体可直接输出带评分的7S诊断报告，而非仅列出7个维度的概念。用户可基于评分快速识别组织薄弱点，龙爪输出从"定性描述"升级为"定量+定性双重诊断"。


### P0-3：升级五行人格适配——增加阴阳面分析与数据驱动识别

**⚡ 龙爪实测证据**

堂食创业单元7S诊断报告中，三店店长的五行识别**全部基于经营数据和行为数据**，而非自报类型：金月湾148.8%超额→"目标强执行快"→金行；松山湖50%渗透率→"销售热情高"→火行。且每店都有**阳面3-4项 + 阴面2-3项**的阴阳面标签（如：✅目标感强 ❌关系维护弱）。

当前岗位智能体Step 2仅有5种五行类型的一维适配表（核心特质→适配岗位→适配任务），**没有阴阳面分析**，且假设用户自报五行类型，**没有数据驱动识别机制**。

**🔧 具体修改**

**升级 Step 2：五行人格识别与适配 → 五行人格深度诊断**

1. 新增"数据驱动五行识别"机制——当用户无法自报五行类型时，从行为/数据特征自动推断
2. 新增"五行阴阳面分析表"——每种五行的阳面3-5项优势 + 阴面3-5项风险，附行为证据模板
3. 新增"阴面警示话术"——当识别到阴面时，龙爪输出警示文字
4. 保留原有一维适配表作为"基础层"，阴阳面分析作为"深度层"

```yaml
# 升级 Step 2：新增深度层
五行阴阳面分析表:
  金行人:
    阳面（显发）:
      - 目标导向极强: [行为证据模板]
      - 执行速度极快: [行为证据模板]
      - 原则清晰公正: [行为证据模板]
      - 数据意识强: [行为证据模板]
    阴面（过刚）:
      - 团队沟通偏指令式: [行为证据模板]
      - 文化把关弱: [行为证据模板]
      - 弹性排班不足: [行为证据模板]
      - 过程指标少: [行为证据模板]
      - 复盘深度浅: [行为证据模板]
    阴面警示话术: "金行过刚——用数量忙碌掩盖质量溃败，做更多不如做更深"
  # 木/火/土/水类似设计

数据驱动五行识别机制:
  触发条件: 用户未提供五行类型 或 用户提供的数据与自报类型矛盾
  识别逻辑:
    - 目标导向+执行快+数据驱动 → 金行信号
    - 热情高+感染力强+跟进弱 → 火行信号
    - 创新力+前瞻性+落地弱 → 木行信号
    - 稳健+流程强+变革慢 → 土行信号
    - 灵活+策略强+定力弱 → 水行信号
  输出: 推断的五行类型 + 识别依据 + 阴阳面标签
```

**📈 预期效果**

龙爪不再依赖用户自报五行类型，可从行为数据自动推断。阴阳面分析让诊断从"你适合什么"升级为"你的强项和暗礁分别是什么"，直接支撑复盘画布和拔阴取阳方案。


### P0-5：新增总部赋能接口激活状态评估机制

**⚡ 龙爪实测证据**

堂食创业单元7S诊断报告中，5大总部赋能接口各有**激活状态评估**：数据赋能=已激活、培训赋能=部分激活、资源赋能=待加强、问题赋能=薄弱、反馈赋能=未激活。每个接口还有✅已完成项和❌缺失项的具体列举。文末有龙爪判断："5大接口中仅数据赋能基本到位，反馈赋能完全空缺"。

当前岗位智能体和味藏店长龙爪SKILL.md都**定义了5大接口的功能**，但**都没有激活状态评估方法论**——如何判断某个接口是"已激活"还是"未激活"？评估标准是什么？证据来源是什么？

**🔧 具体修改**

**新增 Step 4.5：总部赋能接口激活状态评估**

1. 新增5级激活状态定义（已激活/部分激活/待加强/薄弱/未激活）
2. 新增每个接口的激活评估标准——哪些能力项达标=已激活
3. 新增"已完成/缺失"清单模板
4. 新增"接口激活判断"输出格式

```yaml
# 新增 Step 4.5：总部赋能接口激活状态评估
5级激活状态:
  已激活: 核心能力项100%可用，门店可直接调用
  部分激活: 部分能力可用，存在关键缺失项
  待加强: 基础能力在建设中，核心流程未跑通
  薄弱: 仅有个别触点，系统性支撑不足
  未激活: 该接口完全空缺，无任何可用能力

各接口激活评估标准:
  数据赋能:
    已激活: 营业额+成本+客诉数据可获取
    缺失信号: 品控数据/满意度/培训考核数据缺失
  培训赋能:
    部分激活: 有标准化培训流程
    缺失信号: 话术素材/实操模板/深度培训缺失
  # ...其他3个接口类似

输出格式:
  每个接口:
    状态: [5级之一]
    ✅ 已完成: [列举]
    ❌ 缺失: [列举]
  龙爪判断: [综合评估 + 优先打通建议]
```

**📈 预期效果**

龙爪不再仅"描述接口功能"，而能"评估接口是否生效"。用户可直观看到总部赋能的缺口在哪里，优先打通哪个接口。


### P0-7：新增HTML输出格式与视觉设计规范

**⚡ 龙爪实测证据**

两份HTML都采用了悟空偏好的**深色顶栏 + 白色画布 + 浅灰网格**布局，包含：CSS变量体系（--topbar-bg/--canvas-bg/--grid-bg等）、响应式设计（@media断点）、进度条动画（setTimeout渐入）、Tab切换（JS函数switchCal）、颜色编码标签体系（tag-g/tag-r/tag-o/tag-b/tag-p）。

当前岗位智能体**完全没有视觉输出规范**。龙爪模式输出模板仅有Markdown结构定义，没有HTML/CSS/JS规范。龙爪每次输出都需要AI"凭感觉"写HTML，导致风格不统一、布局不一致。

**🔧 具体修改**

**新增 附录A：龙爪HTML输出视觉设计规范**

1. 新增CSS变量体系定义（必须遵循）
2. 新增通用组件库：卡片/表格/标签/进度条/仪表盘卡片/Tab切换
3. 新增响应式断点规范
4. 新增颜色编码逻辑（绿=达标/橙=关注/红=预警/蓝=过程/紫=特殊）
5. 新增交互组件规范（Tab切换、进度条动画）

```yaml
# 新增 附录A：HTML输出视觉设计规范
CSS变量体系（必须遵循）:
  --topbar-bg: #1e293b 或 #1a1a2e（深色顶栏）
  --canvas-bg: #ffffff（白色画布）
  --grid-bg: #f5f6f8 或 #f5f5f4（浅灰网格）
  --accent: #c0392b（警示红）
  --gold: #d4a017（金标色·龙爪品牌色）
  --green: #27ae60（达标绿）
  --blue: #2980b9（过程蓝）
  --orange: #e67e22（关注橙）
  --purple: #8e44ad（特殊紫）

颜色编码逻辑:
  绿(tag-g): 达标/完成/优势
  红(tag-r): 预警/未达标/薄弱
  橙(tag-o): 关注/偏弱/进行中
  蓝(tag-b): 过程/监控/信息
  紫(tag-p): 特殊/战略/文化

通用组件:
  - 顶栏(.topbar): 深色背景+白色文字+sticky定位
  - 卡片(.card): 白色背景+圆角+阴影+1px边框
  - 进度条(.progress-bar): 6-8px高+圆角+渐变填充+过渡动画
  - 仪表盘卡片(.dash-card): 网格布局+图标+大数字+副文本
  - Tab切换(.cal-tab/.cal-panel): JS切换+active样式

响应式断点:
  @media (max-width: 900px): 4列→2列
  @media (max-width: 600px): 全部→1列
```

**📈 预期效果**

龙爪HTML输出风格统一、组件复用、响应式一致。AI无需每次"凭感觉"写CSS，遵循规范即可生成符合品牌调性的报告。


### P1-8：升级经营分析框架——增加多门店对比与经营仪表盘

**⚡ 龙爪实测证据**

堂食创业单元7S诊断报告中有**8张仪表盘卡片**（三店总销售/综合完成率/高价值销售/风险门店/三店分别/预警），支持**三店同时对比**（完成率93.9%~148.8%的跨店比较）。

当前岗位智能体有"经营分析框架"但仅有yaml字段定义，**没有多门店对比模板**，**没有仪表盘卡片布局规范**。假设的是单店/单岗位场景。

**🔧 具体修改**

1. 经营分析框架新增"多门店/多单元对比"模式
2. 新增"经营仪表盘"卡片布局模板（4列网格，每卡片=图标+标签+大数字+副文本）
3. 新增"风险门店/薄弱单元"自动识别逻辑——完成率最低的自动标红
4. 新增"预警卡片"模板——背景色变化（#fde8e8）+预警内容


### P1-10：升级三层行动日历——增加攻坚标记与新增项标注

**⚡ 龙爪实测证据**

堂食创业单元7S诊断报告中的三层行动日历，每日清单有**【攻坚】标记**（品控抽检松山湖）和**【新增】标记**（1对1邀约确认、话术演练时间、门店反馈表填写），且用颜色区分（攻坚=红色、新增=蓝色）。每月规划有**【硬截止】标记**。

当前味藏店长龙爪的三层行动日历**仅有时段+事项+勾选框**，没有攻坚/新增/硬截止标记，无法区分"常规动作"与"重点攻坚"。

**🔧 具体修改**

1. 行动日历新增3种标记：【攻坚】红色 + 【新增】蓝色 + 【硬截止】红色粗体
2. 新增"标记注入逻辑"——P0风险指标→每日清单中自动注入攻坚项；上周缺失项→自动注入新增项；有截止日的任务→自动注入硬截止项
3. 每日清单模板从4时段细化为6时段（新增14:30-15:00话术演练时段、21:00收店后增加反馈表填写）

### P2-13：重构龙爪模式SOP流程——从8步到10步

**⚡ 龙爪实测证据**

本次反推共发现**6个需要新增/插入的步骤**（Step 2.5/3.5/4.5/7.5 + 附录A），加上原有8步，实际流程已超过10步。但当前的"Step X"编号方式在插入后会导致编号混乱。

建议将龙爪模式的SOP从"线性8步"重构为"分层10步"，明确区分核心流程和增强流程。

**🔧 具体修改**

重构后的10步SOP：

**核心流程（必走·6步）：**

| Step | 内容 | 变化 |
|------|------|------|
| Step 1 | 六维信息采集 | 不变 |
| Step 2 | 五行人格深度诊断 | **升级**：+阴阳面+数据驱动识别 |
| Step 3 | 7S组织诊断评分 | **新增** |
| Step 4 | 三层指标体系+关键动作设计 | **升级**：分层+P0/P1 |
| Step 5 | AI增强关键词选择 | 不变 |
| Step 6 | 选择输出类型 | **升级**：+B1/B2双模板 |

**增强流程（按需·4步）：**

| Step | 内容 | 变化 |
|------|------|------|
| Step 7 | 复盘画布生成 | **新增**：B1个人复盘专属 |
| Step 8 | 总部赋能接口评估+心文化评估 | **新增**：B2组织诊断专属 |
| Step 9 | 资产引用填充+龙爪断语生成 | **升级**：+龙爪断语 |
| Step 10 | Rule文件生成+HTML视觉规范校验 | **升级**：+视觉规范校验 |
